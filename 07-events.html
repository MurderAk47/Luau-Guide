Events ⚡

Roblox is event-driven. Instead of constantly checking "did someone touch this brick?", you connect a function that fires automatically when something happens. A player joins? Fire. A brick gets touched? Fire. Someone clicks? Fire. This is how all real Roblox games work.

🔌 How :Connect() works

Every event has a :Connect() method. You pass it a function, and that function runs every time the event fires:

connect.lua
local part = game.Workspace.MyPart
 
part.Touched:Connect(function(hit)
-- "hit" = whatever touched the part
print("Touched by: " .. hit.Name)
end)
 
-- This function runs automatically
-- every time something touches the part!
💡
You don't call the function yourself — Roblox calls it for you when the event happens. That's the whole point of events!

👥 Player events

players.lua
-- Fires when a player joins
game.Players.PlayerAdded:Connect(function(player)
print(player.Name .. " joined the game!")
end)
 
-- Fires when a player leaves
game.Players.PlayerRemoving:Connect(function(player)
print(player.Name .. " left the game.")
end)

🧱 Real example: a kill brick

Here's a complete, real working script you can use in your game right now:

killbrick.lua
local killBrick = game.Workspace.KillBrick
 
killBrick.Touched:Connect(function(hit)
-- Find the Humanoid inside the character
local humanoid = hit.Parent:FindFirstChild("Humanoid")
 
-- Only kill if it's actually a player
if humanoid then
humanoid.Health = 0
end
end)
✅
We check for a Humanoid first because the part fires Touched for everything — not just players. Checking for Humanoid makes sure we only kill actual characters.
