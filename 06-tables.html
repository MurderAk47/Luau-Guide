Tables 📋

A table is like a backpack. Instead of having 10 separate variables for 10 items, you put them all in one table and access them by position or name. Tables are how Luau handles lists, inventories, player data, and pretty much everything complex.

📝 Array — a numbered list

An array stores items in order, numbered starting at 1 (not 0!):

array.lua
local weapons = {"Sword", "Bow", "Staff"}
 
print(weapons[1]) -- Sword
print(weapons[2]) -- Bow
print(weapons[3]) -- Staff
 
-- Add a new item
table.insert(weapons, "Wand")
print(#weapons) -- 4 (count items)
⚠️
Starts at 1! In Luau, the first item is at index 1, not 0 like in JavaScript or Python. This is the #1 source of confusion for beginners.

🗂️ Dictionary — named keys

A dictionary stores data as key → value pairs, like a player profile:

dictionary.lua
local player = {
name = "Zack",
health = 100,
level = 5,
isAlive = true
}
 
print(player.name) -- Zack
print(player.health) -- 100
 
player.health = player.health - 20
print(player.health) -- 80

🔄 Looping through tables

tableloop.lua
-- ipairs: for ordered arrays (keeps order)
local fruits = {"Apple", "Banana", "Mango"}
for index, fruit in ipairs(fruits) do
print(index .. ": " .. fruit)
end
 
-- pairs: for dictionaries (any order)
local stats = {hp = 100, mp = 50, atk = 30}
for key, value in pairs(stats) do
print(key .. " = " .. value)
end
