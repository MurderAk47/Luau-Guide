Variables 📦

A variable is like a labeled box. You put something inside it, give it a name, and later you can open that box and use whatever's in it. Variables are how your script remembers information — like a player's name, their score, or whether they're alive.

📝 Creating a variable

You always start with the word local, then the name you want, then = and the value.

variables.lua
local playerName = "Zack" -- a name (text) local score = 0 -- a number local isAlive = true -- true or false print(playerName) -- Zack print(score) -- 0 print(isAlive) -- true
💡
Always use local! Writing local before a variable keeps it organized and scoped. It's the correct way in Luau — get into the habit now.

🔢 The 4 types of values

Every variable holds one of these types of data:

String — text
"Hello" or 'Hello'
Always wrapped in quotes
Number
42 or 3.14
No quotes — just the number
Boolean — yes/no
true or false
Only ever one of two values
Nil — nothing
nil
Means "this doesn't exist"

🔄 Updating a variable

Once a variable exists, you can change what's inside it — just don't write local again:

update.lua
local score = 0 print(score) -- 0 score = score + 10 -- add 10 print(score) -- 10 score = score + 5 -- add 5 more print(score) -- 15
✅
After you create a variable with local, just use its name to update it. Don't write local again or you'll accidentally create a second variable!

🔗 Combining text with ..

Two dots .. glue text together. This is called concatenation.

concat.lua
local name = "Zack" local score = 99 print("Player: " .. name) -- Player: Zack print("Score: " .. score) -- Score: 99 print(name .. " got " .. score) -- Zack got 99
